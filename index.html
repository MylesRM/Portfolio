Capstone Project

My capstone project, ShiftLite, is a mobile management application designed to improve operational efficiency in fast-paced hospitality environments.

Project Summary

ShiftLite is a mobile management application created to improve day-to-day operational efficiency in hospitality settings. The system is designed to help staff and managers quickly access information, complete essential tasks, and return to their work with minimal interruption.

Project Vision

The vision of ShiftLite is to provide a lightweight, high-efficiency management tool that prioritizes speed, clarity, and usability. Instead of relying on communication-heavy workflows, the app supports short, task-focused interactions that can typically be completed in under one minute.

Business / System Requirements

  • Role-based user authentication for staff and managers
  • Task and table management features
  • Inventory tracking with filtering, sorting, and search
  • Role-based dashboards for quick decision-making
  • High-visibility alerts and notifications
  • Secure handling of user information

Requirements Analysis & Design

The system was designed around speed and usability. Key design choices include simple role-based dashboards, minimal interaction flows, and quick-access information views. The interface was planned to reduce navigation time and make important information immediately visible and actionable.

Wireframes / Mockups

Wireframes and UI mockups were designed using Figma to plan the layout and user experience of the application. These designs include dashboards, task management screens, inventory interfaces, and navigation flows, ensuring that all interactions remain fast, clear, and easy to use before development.

View Figma Designs

System Implementation

ShiftLite is being implemented as a mobile application with features including role-based authentication, real-time communication support, inventory management, scheduling tools, and dashboard functionality. Development follows an iterative sprint-based process so the system can be refined over time.

Project Plan / Timeline

Planning Phase

Research, requirements gathering, user stories, project summary, project vision, and documentation were completed as part of the planning stage.

Development Phase

Core implementation includes environment setup, database configuration, authentication and role systems, real-time communication, inventory management, scheduling, and dashboard development.

Testing Phase

Upcoming work includes integration testing, user acceptance testing, bug fixing, optimization, and post-event reporting.
